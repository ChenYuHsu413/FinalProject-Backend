# AI SERVO PLATFORM — 後端資料規格書 (Backend)

> 給 Backend 團隊：本文件定義你須實作的 API 端點、WebSocket 推播、模型服務、資料儲存與排程。

---

## 一、後端整體職責

```
你負責:
  1. L1 模型服務 — 載入 LightGBM，50kHz 即時推理，每秒聚合
  2. REST API — 13 個端點供 Frontend 呼叫
  3. WebSocket — 7 個 topic 即時推送給 Frontend
  4. 模型版本管理 — models.jsonl append-only 日誌
  5. Fallback 鏈 — NaN/Inf 偵測、前值沿用、PID 切換、hash chain 合規
  6. 資料管線排程 — L2 (1min)、L3 (4HR)、SHAP (觸發時)
```

## 三、WebSocket 實作規格

### 3.1 連線管理

- 端點：`ws://{host}/ws/{topic}`
- Frontend 訂閱特定 scenario：`ws://{host}/ws/l1/summary?scenario_id=01_Pick_and_Place`
- 斷線自動重連 (client side)，後端保留最近 5 筆訊息供重連後補發

### 3.2 Topic 定義與推播格式

| Topic | 推送內容 | 頻率 | 觸發條件 |
|-------|---------|------|---------|
| `ws/l1/inference` | `{"DV": 0.13, "ylabel": "LN", "confidence": 0.97, "latency_ms": 0.20}` | 1kHz (降採樣) | 每次推理完成 |
| `ws/l1/summary` | `{"DV_mean": 0.13, "ylabel_mode": "LN", "latency": {...}, "fallback_count": 0}` | 1s | 每秒聚合完成 |
| `ws/l2/finetune` | `{"rmse_before": 0.042, "rmse_after": 0.038, "improvement_pct": 9.5, "rollback": false}` | 1min | L2 微調完成 |
| `ws/l3/deploy` | `{"algorithm": "LightGBM_HighAcc", "version": "v1.0.3", "decision": "DEPLOY"}` | 4HR | Shadow 通過 |
| `ws/shap/diagnosis` | `{"severity": "high", "top_cause": "FE_RMS", "recommended_mode": "FineTune"}` | 事件時 | SHAP 完成 |
| `ws/fallback/event` | `{"reason": "NaN_output", "level": 1, "action": "use_previous"}` | 事件時 | Fallback 發生 |
| `ws/fallback/escalation` | `{"trigger": "consecutive_3", "control_mode": "PID_BACKUP", "alert": {...}}` | 升級時 | 連續 3 次 Fallback |
| `ws/control/status` | `{"system_state": "RUNNING", "training_state": "INFERENCE_ONLY", "alarm_count": 0}` | 1s | 主控制台狀態 |
| `ws/expert/notification` | `{"event_id": "exp-...", "action": "manual_parameter_adjust", "operator": "工程師 A"}` | 事件時 | 專家介入通知 |

## 四、模型服務實作

### 4.1 L1 模型服務 (50kHz 即時推理)

```python
# 流程 (你需實作):
class L1InferenceService:
    def __init__(self):
        self.model = joblib.load("L1_LightGBM.pkl")  # LightGBM 優先
        self.classifier = joblib.load("L1_LightGBM_cls.pkl")
        self.label_encoder = joblib.load("L1_label_encoder.pkl")
        self.buffer = RingBuffer(50000)  # 1 秒 buffer

    def predict(self, fe, vel, acc, torque, id, iq):
        """每 0.02ms 呼叫一次"""
        X = np.array([[fe, vel, acc, torque, id, iq]])
        t0 = time.time()
        dv = self.model.predict(X)[0]
        cls_proba = self.classifier.predict_proba(X)[0]
        ylabel = self.label_encoder.inverse_transform([np.argmax(cls_proba)])[0]
        confidence = float(np.max(cls_proba))
        latency = time.time() - t0
        self.buffer.append({"dv": dv, "ylabel": ylabel, "latency": latency})
        return dv, ylabel, confidence, latency

    def get_second_summary(self):
        """每秒呼叫一次，產生 ws/l1/summary 內容"""
        data = self.buffer.flush()
        return {
            "DV_mean": np.mean([d["dv"] for d in data]),
            "DV_std": np.std([d["dv"] for d in data]),
            "ylabel_mode": mode([d["ylabel"] for d in data]),
            "latency": {
                "mean_ms": np.mean([d["latency"] for d in data]) * 1000,
                "p99_ms": np.percentile([d["latency"] for d in data], 99) * 1000,
            }
        }
```

### 4.2 L2 微調服務 (每分鐘)

- 從 in-memory buffer (最近 5 分鐘 = 15M samples) 取資料
- 自行 sliding window → 統計特徵
- 載入基底 MLP，凍結 fc1/bn1/fc2，微調 fc3+out
- 每步評估 VAL RMSE，惡化即 rollback
- 完成後推播 `ws/l2/finetune`

### 4.3 L3 AutoML 排程 (每 4HR)

- 從 Stage 4 Parquet 載入資料
- 執行完整 AutoML 矩陣 (含 LightGBM 4 變體)
- 選擇 champion (LightGBM 優先策略)
- 備份舊模型到 L3_archive/
- 觸發 Stage 6 Shadow Mode
- 推播 `ws/l3/deploy`

## 五、Fallback 鏈實作

```python
# 你需實作的 Fallback Chain:
class FallbackChain:
    def __init__(self):
        self.prev_dv = None
        self.prev_ylabel = None
        self.consecutive_count = 0
        self.pid_kp, self.pid_ki, self.pid_kd = 1.5, 0.05, 0.01

    def process(self, dv, ylabel):
        if dv is None or ylabel is None or math.isnan(dv) or math.isinf(dv):
            self.consecutive_count += 1
            if self.consecutive_count == 1:
                return self.prev_dv, self.prev_ylabel, "use_previous"
            elif self.consecutive_count >= 3:
                return self._pid_backup(), "PID_BACKUP", "switch_to_PID"
        else:
            self.consecutive_count = 0
            self.prev_dv = dv
            self.prev_ylabel = ylabel
            return dv, ylabel, "normal"

    def _pid_backup(self):
        # PID 備份公式 (簡化)
        return self.prev_dv * self.pid_kp
```

## 九、控制模式狀態機（補充）

### GET `/api/v1/control-mode`
**用途：** 當前控制模式狀態機

**Query Params:**
- `scenario_id` (required)

**Response 200:**
```json
{
  "scenario_id": "01_Pick_and_Place",
  "current_mode": {
    "code": 0,
    "name": "Normal",
    "hmi_color": "green"
  },
  "mode_history": [
    {"from": null, "to": "Normal", "at": "2026-07-23T00:00:00Z", "trigger": "system_start"},
    {"from": "Normal", "to": "Diagnosis", "at": "2026-07-23T01:50:00Z", "trigger": "residual_exceed_3cycles"},
    {"from": "Diagnosis", "to": "Normal", "at": "2026-07-23T02:00:00Z", "trigger": "repair_confirmed"}
  ],
  "state_machine": {
    "Normal → Diagnosis":   "殘差連續 > 閾值 × 3 個週期 (自動觸發)",
    "Diagnosis → Normal":   "操作員確認維修或忽略後殘差恢復正常",
    "Normal → FineTune":    "操作員手動切換 (性能優化需求)",
    "FineTune → Normal":    "優化完成或超時",
    "any → Safe":           "Fallback 連續 3 次或 DV > 0.8"
  }
}
```

**控制模式轉換規則（對照 AI SERVO PLATFORM 文件）：**
```
Normal (0):     輕量模型推理 + EtherCAT 回饋，SHAP 後臺運行 (異常時報警)
  │
  │ 殘差連續 > 閾值 × 3 個週期 (自動)
  ▼
Diagnosis (1):  暫停控制優化 (保持當前輸出)，SHAP 深入分析故障根因，生成維修建議
  │
  │ 操作員確認維修 → 殘差恢復正常
  ▼
Normal (0)      或操作員手動切 → FineTune (2)
  │
  │ 性能優化需求 (手動)
  ▼
FineTune (2):   輕量模型 + 閉環微調 (無 SHAP)，SHAP 暫停只記錄資料
  │
  │ 優化完成或超時
  ▼
Normal (0)
```

## 十三、排程時程（完整版）

| 任務 | 週期 | 觸發方式 | 預估時間 |
|------|------|---------|---------|
| L1 推理 | continuous (50kHz) | 每 sample | <0.5ms |
| L1 聚合 + WebSocket 推播 | 1s | 計時器 | <10ms |
| L2 微調 | 1min | 計時器 | <30s |
| L3 AutoML | 4HR | 計時器 | ~2HR |
| SHAP 診斷 | 事件觸發 (DV>0.4) | L1 異常偵測 | ~5min |
| Fallback 處理 | 事件觸發 (NaN/Inf) | L1 推理異常 | <1ms |
| 資料清理 (舊 parquet) | 24HR | 計時器 | 視資料量 |
